# UX/UI References

These references informed the first design exploration. They are not copied into the app; they shape layout decisions and QA criteria.

- Material 3 navigation bar: use bottom navigation for a small set of top-level mobile destinations.
  - https://m3.material.io/components/navigation-bar/overview
- Material 3 navigation rail: use a compact rail for medium and large screens.
  - https://m3.material.io/components/navigation-rail/overview
- Apple Human Interface Guidelines, navigation and search: navigation should feel familiar and support the content rather than dominate it.
  - https://developer.apple.com/design/human-interface-guidelines/navigation-and-search
- web.dev PWA architecture: app-shell architecture and progressive enhancement should follow the app's constraints.
  - https://web.dev/learn/pwa/architecture
- W3C WCAG 2.2: target size and visible focus remain part of implementation QA.
  - https://www.w3.org/TR/WCAG22/
